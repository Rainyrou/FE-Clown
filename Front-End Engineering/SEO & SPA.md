SEO：

* 配置 `robots.txt` 协议让爬虫可访问首页、核心页面和站点地图并通过内链相互关联，扁平化站点结构减少爬取深度
* HTML 语义化标签，优化 `title/meta` 标签与链接文本，结构化数据 Schema 标记
* 图片添加语义化 `alt` 属性，视频添加 `track` 标签和 `title` 属性
* 多端适配 + 首屏优化确保爬虫抓取完整内容如通过 SSR 预渲染静态页面或 SSG 提前构建静态页面

SPA：一次性加载 HTML，通过 JavaScript 动态渲染 DOM 与路由（客户端动态渲染替代服务端多页面渲染），解决 MPA 页面跳转刷新的体验问题

* 首屏：SPA 首次请求返回空 HTML `<div id="root"></div>`，浏览器需完成加载解析 JavaScript Bundle（首屏组件 + 前端框架+ 全局状态 + 路由库）-> 初始化前端框架 -> 请求首屏所需数据 -> 浏览器渲染后才显示首屏内容，导致 FCP 和 TTI 显著延迟
* SEO：SPA 动态渲染机制与搜索引擎"抓取 - 解析"逻辑存在冲突，传统爬虫仅能读取服务端返回的空 HTML，虽然现代爬虫支持执行 JavaScript ，但其仍需等待静态资源加载和 JavaScript 执行，且对单页面 JavaScript 执行时间限制为 3-5 秒，部分 SPA 未完成渲染，导致未抓取到完整页面
* 内存：SPA 单页面不刷新特性导致 JavaScript 上下文、Window 全局对象和全局作用域生命周期长，若资源（全局变量 + 闭包引用 + 事件监听器和定时器 + 第三方库实例）未及时释放导致内存持续增长，页面崩溃
