微前端架构将大型单体应用拆分为多个独立开发和部署的子应用，在运行时无缝集成于同一整体

- 模块联邦：构建工具的模块联邦插件让主应用在运行时动态加载子应用代码，通过子应用暴露自身模块和共享第三方库依赖实例实现依赖复用并版本统一
- 沙箱隔离：qiankun 通过 Proxy 代理浏览器 `window` 对象并创建 `fakeWindow`，子应用在 `fakeWindow` 中增删修改属性，Shadow DOM + 编译阶段添加 CSS 类名前缀 + 运行阶段样式作用域标识实现样式隔离、路由前缀匹配机制实现路由隔离、动态资源绑定 & 移除 + 全局事件注册 & 移除实现资源与事件隔离
- 依赖管理 & 依赖共享：包管理工具通过依赖提升、硬链接或符号链接实现依赖管理，构建工具的 Dev Server 转发主子应用间的请求响应实现子应用模块的实时共享
- 预加载：主应用预加载并缓存子应用可能的公共依赖以避免请求瀑布（主应用加载 -> 子应用加载 -> 子应用公共依赖加载）
