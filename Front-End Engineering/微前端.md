微前端架构将大型单体应用拆分为多个独立开发和部署的子应用，在运行时无缝集成于同一整体

- 模块联邦：Webpack5 模块联邦插件让应用在运行时动态加载其他应用代码，通过暴露配置和共享模块实现应用间的依赖复用，主应用定义从子应用加载的模块，子应用暴露自身模块供主应用按需加载，共享依赖为多应用间共享第三方库依赖实例，避免重复加载并强制版本统一
- 沙箱隔离：qiankun 通过 Proxy 代理浏览器 `window` 对象创建 `fakeWindow`，子应用在 `fakeWindow` 中增删修改属性，通过 Snapshot 沙箱（全量快照 + 还原）和 Legacy 沙箱（状态 diff + 备份还原）兼容旧版本浏览器、Shadow DOM + 编译时样式前缀 + 运行时样式作用域标记 + 样式表激活 & 卸载实现样式隔离、路由前缀匹配机制实现路由隔离、动态资源绑定 & 移除 + 全局事件拦截 & 解绑实现资源与事件隔离
- 依赖管理 & 依赖共享：包管理工具通过依赖提升、硬链接或符号链接实现依赖管理，构建工具的 Dev Server 转发主子应用间的请求和响应以实现子应用模块实时共享
- 预加载：主应用预加载子应用可能的公共依赖并将其缓存以避免请求瀑布（主应用加载 -> 子应用加载 -> 子应用公共依赖加载）
