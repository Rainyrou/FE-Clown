暗黑主题：浏览器启动时读取操作系统的颜色偏好设置并将其映射为 `prefers-color-scheme`，前端通过媒体查询 `prefers-color-scheme`  与 JavaScript `window.matchMedia/addEventListener` 监听主题变化

- 样式覆盖：通过在 HTML 标签上设置两套主题 CSS 类名
- CSS 变量：在根元素上设置两套主题 CSS 变量，生成主题无关的变量名 -> 主题相关的变量值的映射表，修改根元素 CSS 变量值即可切换主题，依赖其的组件样式自动更新

| 变量名（主题无关） | 亮色模式值 | 暗黑模式值 |     |
| ------------------ | ---------- | ---------- | --- |
| `--color-bg`       | #ffffff    | #121212    |     |
| `--color-text`     | #333333    | #e0e0e0    |     |
| `--color-primary`  | #2196F3    | #4DABF7    |     |

- token：分为基础 token（纯颜色值，无业务含义如 `--color-white: #ffffff`） → 语义化 token（关联业务含义，修改语义化 token 的映射即可切换主题如 `--color-bg-page: var(--color-white)`） → 组件 token（组件样式，依赖语义化 token，无需单独适配如 `--button-bg-primary: var(--color-primary)`），无需为各个组件单独编写，减少 CSS 代码量
- 兼容性：样式隔离与继承 + 解决样式冲突 + 强制覆盖
* 存储与同步：优先级用户手动设置（存储于 `localStorage` 或 `IndexedDB`） > 系统主题（通过 JavaScript `window.matchMedia/addEventListener` 读取） > 应用默认（亮色主题）
