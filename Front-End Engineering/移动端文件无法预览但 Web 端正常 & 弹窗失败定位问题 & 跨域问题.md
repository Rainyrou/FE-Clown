- 全链路验证：弹窗前端渲染链路为触发事件 → 状态更新 → DOM 渲染 → 交互响应，从事件绑定 + 事件流 + 触发条件的角度看，事件需成功绑定至 DOM 元素，未被阻止冒泡和默认行为或被更高优先级事件覆盖且需触发弹窗条件，从状态管理的角度看，状态变化需触发组件重新渲染，正确更新弹窗显隐状态，从 DOM 渲染的角度看，DOM 树需存在弹窗节点，样式需可见且未被覆盖，从事件循环的角度看，渲染线程和异步操作需正常执行，从前后端交互角度看，前端请求和后端数据需正常发送和返回，返回数据需符合预期，从环境角度看，机型设备、操作系统和浏览器需兼容相关 API，环境资源需配置正确和加载成功

接口访问正常但前端无法获取数据 + 无 OPTIONS 请求，定位问题：

- 检查请求是否为简单请求如 GET、HEAD 或 POST 请求且请求头部符合 W3C 标准定义的 CORS 安全请求头部集合
- 检查响应头部字段  `Access-Control-Allow-Origin/Access-Control-Allow-Credentials/Access-Control-Expose-Headers` 是否存在 & 配置正确且与请求头部字段 `Origin/withCredentials` 是否匹配
- 检查网关是否配置 CORS 规则及是否覆盖后端配置，移除 CDN 和浏览器缓存
- 通过浏览器控制台打印 CORS 拦截错误，根据错误信息直接定位问题
