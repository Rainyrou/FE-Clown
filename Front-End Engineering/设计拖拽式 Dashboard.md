- 结论：我认为，设计拖拽式 Dashboard 的本质为构建一个轻量级的低代码渲染引擎，核心为 Schema First，将视图、数据和逻辑解耦为 DSL，渲染引擎只负责解析 Schema，实现配置即运行
- 核心逻辑：视图层面，基于 Grid 网格布局算法，维护 Layout Schema （x, y, w, h），多端适配，并处理拖拽碰撞和自动吸附问题；渲染层面，维护动态组件注册表， 通过 type 字段映射组件，实现组件的懒加载和实例化；数据层面，后端数据格式通过适配层统一转换为标准格式，再传递给图表组件
- 落地细节：实战中，重点解决两个性能痛点： 第一，重绘性能，设计交互态冻结机制，配合 `ResizeObserver`，仅在图表停止拖拽且尺寸变化时触发 `resize` 计算；第二，组件联动，设计基于声明式的发布订阅机制，组件在 DSL 中配置订阅规则，如折线图订阅了时间筛选器，当筛选器变化时，EventBus 自动通知折线图更新
