- 架构核心：上层无感，底层适配，上层为统一 DSL 与 API 抽象，底层为面向多端的编译后端和渲染后端
- 样式渲染：C++ 布局引擎 + 原生控件映射，C++ 布局引擎统一计算抹平 Flexbox 差异，计算原生坐标（x, y, w, h），驱动多端原生控件渲染，绝不模拟
- 功能 API：C++ 绑定 + 生命周期抽象，通过 JSI/NAPI 将 C++ 适配层同步注入 JavaScript 运行时，JavaScript 直接调用 C++，C++ 层分发至各端 Native API，同时抽象对齐 iOS/Android/HarmonyOS/小程序生命周期钩子
- 性能优化：JSI 同步调用 + 批量通信 + 原生组件池，通过 C++ 绑定消解 Bridge 异步瓶颈，自动批处理样式渲染与 API 调用，合并为单次跨线程通信，复用原生组件池，避免重复创建
- 兼容性：运行时能力检测 + 降级兜底，启动时即检测宿主环境能力，动态注入 Polyfill 或启用降级，上层 API 保持一致，底层抹平版本差异
- 扩展性：可插拔，新增端只需实现其渲染器与 API 适配器接口，通过反向注入机制，允许多端原生生态无缝接入框架
