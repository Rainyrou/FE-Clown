JavaScript 无类型注解，在一定程度上提高开发效率，但存在访问未定义的属性或调用非函数类型等风险，TypeScript 提供类型系统解决上述问题如严格模式 + 类型注解 + 类型推断 + 泛型 + 接口 + 类型别名 + 类型断言 + 类型守卫 + 内置工具类型等，在编译阶段检查类型以提前发现潜在错误而无需等到运行阶段，减少研发的心智负担，TypeScript 最终转换为 JavaScript ，程序运行阶段性能影响较小

TS 转换为 JS：

- 词法分析：词法分析器根据词法规则扫描源代码文本结构，移除源代码的非语法细节（空格/注释/分号），拆分为 Token
- 语法分析：语法分析器根据语法规则验证 Token 的语法合法性并据此建立 Token 间的嵌套关系，将其转化为 AST
- 类型检查：遍历器 DFS AST，校验类型推断、类型兼容性和语法合法性
- 增删修改：遍历器 DFS AST，当访问某类型节点时调用访问者对象对应的处理函数，增删修改 AST 节点，移除 TS 类型节点（类型注解 + 类型推断 + 泛型 + 接口 + 类型别名 + 类型断言 + 类型守卫 + 内置工具类型等），语法降级，生成最终代码

类型体操：TypeScript 类型体操源于其类型系统，通过严格模式 + 类型注解 + 类型推断 + 泛型 + 接口 + 类型别名 + 类型断言 + 类型守卫 + 内置工具类型等能力完成类型层面任意计算
