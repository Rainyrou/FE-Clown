进程：

- 进程为计算机程序执行状态的抽象，当执行程序时，操作系统为其创建一个新进程
- 进程为 CPU 资源分配的最小单位，操作系统为其分配内存
- 各个进程均有其独立的内存地址，一个进程无法直接访问另一进程，进程有独立的资源如 CPU、内存、磁盘和网络等（通过任务管理器即可查看后台进程列表）
- 进程间通信相对较慢，且创建、销毁和切换进程的开销相对较大
- 进程的生命周期由操作系统控制

线程：

- 线程为进程内部的执行单元，一个进程有一到多个线程
- 线程为 CPU 调度的最小单位
- 进程内的所有线程共享同一内存地址，一个线程可访问另一线程
- 线程间通信相对较快，且创建、销毁和切换线程的开销相对较小
- 线程的生命周期由进程控制

协程：

- 协程为线程内部的执行单元
- 协程为用户态逻辑调度的最小单位，其调度对操作系统不可见
- 线程内的所有协程共享同一内存地址
- 协程间通信相对较快，且创建、销毁和切换线程的开销相对较小
- 协程的生命周期由程序代码或用户态调度器控制

并发：在单核处理器上，通过时间片轮转、上下文切换等机制，多个任务在同一时间段内交替执行

并行：在多核处理器上，通过任务拆分、多核调度、负载均衡等机制，多个任务在同一时间点上同时执行

进程间通信：管道 + 消息队列 + 套接字 + 信号量 + 共享内存

进程切换：当进程执行系统调用或 I/O 操作或时间片用尽或高优先级进程从等待状态转换为就绪状态，操作系统的调度程序保存当前进程状态并将其挂起，加载新进程状态至寄存器中，转移控制权给新进程

死锁：在多线程环境中各个进程/线程均在等待其他成员释放资源，同时占有其他进程/线程所需的资源，导致所有进程/线程无法继续执行

进程管理：

- 创建 & 销毁：操作系统创建进程以响应交互或执行服务，分配唯一 PID 和内存，初始化进程控制块，当进程执行完，操作系统回收其资源
- 进程调度：调度器根据调度算法决定获取资源的进程
- 进程间通信
- 进程切换
- 避免死锁

进程调度算法 & CPU 调度：先来先服务 + 短作业优先 + 轮转调度 + 优先级调度 + 多级反馈队列

进程调度算法用于在多进程间分配有限处理器时间，操作系统的调度器决定进程是否使用处理器和使用期限，在单位时间内完成尽可能多的进程，最大化处理器利用率，避免空闲状态，减少用户等待时间

死锁：

- 互斥条件：至少有一个资源处于非共享模式即一次仅有一个进程可使用的资源
- 不剥夺条件：资源仅能由占有其进程/线程主动释放，无法剥夺
- 占有且等待条件：一个进程/线程至少占有一个资源并等待其他成员释放资源
- 循环等待条件：有两个以上的进程形成头尾相接的循环等待资源关系

避免死锁：

- 破坏互斥条件：尽可能使资源共享
- 破坏不剥夺条件：若已占有某资源的进程/线程请求新资源失败，则释放其当前占有的资源
- 破坏占有且等待条件：一次性分配所有资源，进程/线程在开始执行前请求并获取所需资源
- 破坏循环等待条件：操作系统为资源编号，进程依照序号顺序请求资源，防止循环等待

线程同步：互斥锁 + 读写锁 + 条件变量 + 信号量

终止线程：协作式终止（设置共享终止标志位/中断状态位/线程池状态位 + 任务执行完成）+ 强制终止 `stop` + 强制销毁 `destory` + 强行终止进程/线程组

- 物理内存：计算机硬件实际内存
- 虚拟内存：以硬盘为存储，由操作系统控制，比物理内存大，访问速度比物理内存慢，通过结合使用物理内存和磁盘空间，创建连续内存地址
